# MindHit - 제품 개요

## 한 줄 요약
**브라우징 기록을 AI 마인드맵으로 변환하여, 흩어진 생각의 흐름을 한눈에 파악할 수 있는 크롬 확장 + 웹앱 서비스**

---

## 문제 정의 (Problem)

### 현재 상황
- 리서치/조사를 할 때 수많은 탭을 열고 닫음
- 어떤 페이지를 봤는지, 왜 봤는지 기억이 안 남
- 브라우저 히스토리는 단순 나열일 뿐, 맥락이 없음
- 조사가 끝나면 "내가 뭘 알아냈지?" 정리가 안 됨

### 핵심 Pain Point
> "2시간 동안 리서치했는데, 결국 뭘 알아낸 건지 모르겠다"

---

## 솔루션 (Solution)

### 핵심 가치 제안
1. **시간의 재사용** - 브라우징에 투자한 시간을 의미 있는 결과물로 변환
2. **생각의 시각화** - AI가 자동으로 마인드맵을 생성하여 조사 흐름을 구조화
3. **기록의 자동화** - 시작/종료 버튼만 누르면 나머지는 자동

### 작동 방식
```
[Start 클릭] → 자동 기록 시작
     ↓
[브라우징] → 페이지 방문, 체류 시간, 하이라이팅 감지
     ↓
[Stop 클릭] → 세션 종료
     ↓
[AI 처리] → 마인드맵 자동 생성
     ↓
[결과 확인] → 웹앱에서 타임라인 + 마인드맵 확인
     ↓
[이메일 발송] → 세션 요약 리포트 전송
```

## 제품 구성

| 구성 요소 | 역할 | 특징 |
|----------|------|------|
| **Chrome Extension** | 데이터 수집 | 최소한의 UI (Start/Stop만) |
| **Web App** | 데이터 시각화 & 관리 | 메인 사용자 경험 |
| **Backend Server** | 처리 & 저장 | AI 마인드맵 생성, 이메일 발송 |

---

## 타겟 사용자

### Primary
- **기획자/PM**: 시장 조사, 경쟁사 분석 시 리서치 정리
- **연구자/학생**: 논문, 자료 조사 시 레퍼런스 관리

### Secondary
- **개발자**: 기술 문서, 스택오버플로우 조사 정리
- **일반 사용자**: 구매 결정 전 비교 조사 등

---

## 핵심 차별점

| 기존 도구 | MindHit |
|----------|---------|
| 북마크 - 수동 저장 필요 | 자동 기록 |
| 히스토리 - 단순 나열 | 맥락 있는 마인드맵 |
| 노트앱 - 직접 정리해야 함 | AI 자동 요약 |
| Notion/Obsidian - 학습 곡선 있음 | 버튼 2개로 끝 |

---

## 제품명: MindHit

- **Mind**: 생각, 사고의 흐름
- **Hit**: 방문(페이지 히트), 핵심을 맞추다
- 의미: "당신의 생각 흐름을 정확히 포착한다"
